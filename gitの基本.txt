gitのことが現時点理解できていない基礎から覚える。

ローカル環境だけでも少しは覚えられるけど、リモートを用いてのチーム管理が醍醐味
のようなので、リモートサービスを利用

「github」有名所
・非公開は有料
・公開なら無料で使用できる


githubにアカウント登録し「新規リポジトリ」を作成する。
新規リポジトリー＞新しいプロジェクトみたいなものかな？

作成するとURLが生成されるのでこのあとつかう。


ローカルリポジトリ作成
自分のローカルにgit管理用プロジェクトのフォルダを作成する。

エクスプローラなんかで作成してもいいけど、ぐぐるとターミナルでの作成がおおい。

cd　で作成したいディレクトリに移動して
mkdirでディレクトリを作成して
cd　でそのディレクトリに移動
git init でそのディレクトリがローカルリポジトリ（プロジェクトディレクトリ）になる

これまでが基本


ココからが操作。
リモートリポジトリ（githubのプロジェクト）が新規なら
ローカルでファイルを作成してアップする。

既存なら
 git clone URL
でリモートリポジトリのデータを引っ張ってくる


-------------------ココから追加
ファイルを新規作成したら
git add でインデックス
git commit -m "コメント"でこめんとつける
git remote origin URL でリモートリポジトリを追加

-------------------追記
git remote add origin URL でリモートサーバー登録
git push origin master でまぁアップみないなもの。

-----------------追記
git add,gitcommitは毎回やる。
git add . は全ファイルステージに
git commit -m "1行目" -m "3行目"　となる
